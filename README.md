## 6.贡献代码
>推荐使用以下工作流参与贡献：
> 1. 将仓库 fork 到自己的 GitHub，并 clone 到本地进行开发
> 2. 创建新的分支，在新的分支上开发
> 3. 保持分支与远程 develop 分支一致（通过 fetch 和 rebase 操作）
> 4. 在本地提交变更（注意 commit log 要简要描述提交内容）
> 5. 将提交 push 到 fork 的仓库下
> 6. 创建一个 pull request 到 `dev` 分支
